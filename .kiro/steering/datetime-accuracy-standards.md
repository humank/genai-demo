# 日期時間準確性標準

## 概覽

本文檔定義了在所有文檔、報告和代碼中使用日期時間的準確性標準，確保所有時間戳記都反映實際的撰寫或執行時間。

## 強制性要求

### 📅 日期時間獲取規則

#### 1. 必須使用 MCP Time 工具
在撰寫任何包含日期時間的文檔時，**必須**使用 MCP Time 工具獲取當前準確時間：

```
使用 mcp_time_get_current_time 工具，timezone 參數設為 "Asia/Taipei"
```

#### 2. 禁止假設或猜測日期
- ❌ **絕對禁止**: 基於上下文或記憶假設當前日期
- ❌ **絕對禁止**: 使用過時的日期信息
- ❌ **絕對禁止**: 複製其他文檔中的日期而不驗證

#### 3. 強制驗證機制
每次撰寫包含日期的內容時，必須：
1. 先調用 `mcp_time_get_current_time` 獲取準確時間
2. 使用獲取到的實際日期時間
3. 在文檔中明確標註時區（預設使用台北時間 Asia/Taipei）

## 適用範圍

### 📋 必須包含準確日期時間的文檔類型

#### 報告和摘要文檔
- 所有 `reports-summaries/` 目錄下的文檔
- 評估報告、分析報告、狀態報告
- 會議記錄、決策記錄

#### 架構文檔
- ADR (Architecture Decision Records)
- 設計文檔的最後更新時間
- 版本發布記錄

#### 開發文檔
- 變更日誌 (CHANGELOG)
- 發布說明 (Release Notes)
- 問題追蹤和解決記錄

#### 配置和腳本
- 部署腳本中的時間戳記
- 配置文件的更新時間
- 自動化任務的執行時間

## 日期時間格式標準

### 🕐 標準格式規範

#### 中文文檔格式
```
完整格式: 2025年9月23日 下午2:34 (台北時間)
日期格式: 2025年9月23日
時間格式: 下午2:34
ISO格式: 2025-09-23T14:34:29+08:00
```

#### 英文文檔格式
```
完整格式: September 23, 2025 2:34 PM (Taipei Time)
日期格式: September 23, 2025
時間格式: 2:34 PM
ISO格式: 2025-09-23T14:34:29+08:00
```

#### 技術文檔格式
```
ISO 8601 標準: 2025-09-23T14:34:29+08:00
Unix 時間戳: 1727073269
RFC 3339: 2025-09-23T14:34:29+08:00
```

- [ ] 已調用 `mcp_time_get_current_time` 獲取當前時間
- [ ] 確認時區設定為 "Asia/Taipei"
- [ ] 驗證返回的日期時間格式正確

### ✅ 撰寫中檢查
- [ ] 所有日期時間都使用從 MCP Time 工具獲取的實際時間
- [ ] 日期格式符合文檔類型的標準格式
- [ ] 時區信息明確標註

### ✅ 撰寫後檢查
- [ ] 檢查文檔中所有日期時間的一致性
- [ ] 確認沒有使用假設或過時的日期
- [ ] 驗證時間戳記反映實際的撰寫時間

## 常見錯誤和避免方法

### ❌ 常見錯誤

#### 1. 使用過時日期
```
錯誤示例: "評估日期: 2025年1月22日" (實際是9月23日)
```

#### 2. 假設日期
```
錯誤示例: 基於文檔上下文猜測 "應該是今天"
```

#### 3. 複製舊日期
```
錯誤示例: 從其他文檔複製日期而不驗證
```

#### 4. 缺少時區信息
```
錯誤示例: "2025年9月23日 14:34" (缺少時區)
```

### ✅ 正確做法

#### 1. 每次都獲取實際時間
```
正確流程:
1. 調用 mcp_time_get_current_time
2. 使用返回的實際時間
3. 按標準格式撰寫
```

#### 2. 明確標註時區
```
正確示例: "2025年9月23日 下午2:34 (台北時間)"
```

#### 3. 保持格式一致性
```
在同一文檔中使用統一的日期時間格式
```

## 自動化檢查

### 🔍 Kiro Hook 整合

```json
{
  "name": "DateTime Accuracy Check",
  "description": "檢查文檔中的日期時間準確性",
  "when": {
    "patterns": [
      "**/*.md",
      "**/*.txt"
    ]
  },
  "then": {
    "prompt": "檢查文檔中的所有日期時間是否使用 MCP Time 工具獲取的準確時間，並符合格式標準。"
  }
}
```

### 📊 品質指標
- **準確性**: 100% 的日期時間必須反映實際撰寫時間
- **一致性**: 同一文檔內的日期時間格式必須統一
- **完整性**: 所有需要日期時間的文檔都必須包含準確的時間戳記

## 例外情況

### 🎯 允許的例外情況

#### 1. 歷史記錄
- 記錄過去事件的實際發生時間
- 引用其他文檔的原始日期（需明確標註為引用）

#### 2. 計劃日期
- 未來的計劃或截止日期
- 里程碑和目標日期（需明確標註為計劃日期）

#### 3. 版本控制
- Git 提交時間（由系統自動生成）
- 文件系統的修改時間

### ⚠️ 例外情況處理
即使在例外情況下，也必須：
- 明確標註日期的性質（歷史、計劃、引用等）
- 在可能的情況下提供當前的實際撰寫時間
- 保持格式的一致性

## 責任和問責

### 👥 角色責任

#### 文檔撰寫者
- 必須遵循本標準獲取和使用準確的日期時間
- 負責驗證所撰寫內容中日期時間的準確性
- 在發現錯誤時立即更正

#### 審核者
- 檢查文檔中日期時間的準確性和格式一致性
- 驗證是否使用了 MCP Time 工具獲取的實際時間
- 拒絕包含不準確日期時間的文檔

#### 維護者
- 定期檢查和更新日期時間標準
- 確保 MCP Time 工具的可用性和準確性
- 處理日期時間相關的問題和改進建議
