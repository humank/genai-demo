## Writing Guidelines

### Language Requirements

- **All new documentation must be written in English**
- Use clear, professional technical writing
- Follow consistent terminology and style
- Include proper grammar and technical accuracy

### Structure Requirements

- Use clear headings and sections
- Include table of contents for long documents
- Provide code examples with proper formatting
- Include relevant diagrams and illustrations

### Maintenance Requirements

- Keep documentation up-to-date with code changes
- Review and update documentation regularly
- Use version control for all documentation
- Follow the established file organization structure

For detailed implementation guidelines, please refer to the internal steering documentation.
