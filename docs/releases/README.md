## 發布流程

每次重要更新都應該在本目錄下創建一個新的發布說明文檔，命名格式為：`<主題>-<年份>-<月份>-<日期>.md`。

發布說明應包含以下內容：
1. 業務需求概述
2. 技術實現
3. 架構變更
4. 技術細節
5. 測試覆蓋
6. 結論
