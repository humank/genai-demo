# Domain-Driven Design Patterns\n\n## Overview\n\nThis directory contains tactical pattern implementations and best practices for Domain-Driven Design (DDD).\n\n## DDD Tactical Patterns\n\n### Core Concepts\n- **Aggregate Root** - Entry point and consistency boundary of aggregates\n- **Entity** - Domain objects with unique identity\n- **Value Object** - Immutable descriptive objects\n- **Domain Service** - Business logic that doesn't belong to specific entities\n- **Domain Event** - Important events that occur in the domain\n\n### Implementation Guidelines\n- Aggregate design principles\n- Distinction between entities and value objects\n- When to use domain services\n- Domain event publishing and handling\n\n## Related Documentation\n\n- [Domain-Driven Design](../ddd-domain-driven-design.md)\n- [Domain Events Guide](../../../../../.kiro/steering/domain-events.md)\n- [SOLID Principles](../../solid-principles-and-design-patterns.md)\n\n---\n\n**Maintainer**: Development Team  \n**Last Updated**: January 21, 2025\n![Microservices Overview](../../../../diagrams/viewpoints/development/microservices-overview.puml)\n![Microservices Overview](../../../../diagrams/viewpoints/development/architecture/microservices-overview.mmd)"
