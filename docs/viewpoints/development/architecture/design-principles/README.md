## 核心原則

### 1. 單一職責原則 (SRP)
每個類別應該只有一個變更的理由。

### 2. 開放封閉原則 (OCP)
軟體實體應該對擴展開放，對修改封閉。

### 3. 里氏替換原則 (LSP)
子類別應該能夠替換其父類別。

### 4. 介面隔離原則 (ISP)
不應該強迫客戶端依賴它們不使用的介面。

### 5. 依賴反轉原則 (DIP)
高層模組不應該依賴低層模組，兩者都應該依賴抽象。
