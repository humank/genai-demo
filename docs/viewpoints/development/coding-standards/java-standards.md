# Java 編碼標準

本文檔定義 Java 編碼標準 的編碼規範和最佳實踐。

## 編碼原則

### 可讀性

- 使用有意義的變數和方法名稱
- 保持程式碼簡潔明瞭
- 適當添加註解說明複雜邏輯

### 一致性

- 遵循統一的命名約定
- 使用一致的程式碼格式
- 保持架構模式的一致性

## 編碼規範

### 命名約定

- **類別名稱**：使用 PascalCase
- **方法名稱**：使用 camelCase
- **常數**：使用 UPPER_SNAKE_CASE

### 程式碼結構

- 方法長度不超過 20 行
- 類別職責單一且明確
- 適當使用設計模式

## 品質標準

### 程式碼審查

- 所有程式碼必須經過審查
- 至少需要 2 位審查者
- 修正所有審查意見後才能合併

### 測試要求

- 程式碼覆蓋率 > 80%
- 所有公開方法都有測試
- 包含邊界條件測試
