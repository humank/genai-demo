# Getting Started Guide\n\n## Overview\n\nWelcome to our development team! This guide will help you quickly understand the project architecture, set up your development environment, and make your first code contribution.\n\n## Prerequisites\n\n### Required Software\n- **Java 21** - OpenJDK or Oracle JDK\n- **Node.js 18+** - Required for frontend development\n- **Docker Desktop** - Containerized development environment\n- **Git** - Version control\n\n### Recommended Tools\n- **IntelliJ IDEA Ultimate** - Java development IDE\n- **VS Code** - Frontend development and general editor\n- **Postman** - API testing tool\n- **AWS CLI** - AWS service management\n\n## Environment Setup\n\n### 1. Clone Project\n```bash\ngit clone https://github.com/your-org/genai-demo.git\ncd genai-demo\n```\n\n### 2. Backend Setup\n```bash\n# Check Java version\njava -version\n\n# Build project\n./gradlew build\n\n# Run tests\n./gradlew test\n\n# Start application\n./gradlew bootRun\n```\n\n### 3. Frontend Setup\n\n#### CMC Management System\n```bash\ncd cmc-frontend\nnpm install\nnpm run dev\n```\n\n#### Consumer Application\n```bash\ncd consumer-frontend\nnpm install\nnpm run dev\n```\n\n### 4. Database Setup\n```bash\n# Development environment uses H2 in-memory database, no additional setup needed\n# If you need to use PostgreSQL, refer to docker-compose.yml\n\ndocker-compose up -d postgres\n```\n\n## Project Structure\n\n### Backend Structure\n```\napp/\nâ”œâ”€â”€ src/main/java/solid/humank/genaidemo/\nâ”‚   â”œâ”€â”€ application/          # Application service layer\nâ”‚   â”œâ”€â”€ domain/              # Domain layer\nâ”‚   â”œâ”€â”€ infrastructure/      # Infrastructure layer\nâ”‚   â””â”€â”€ interfaces/          # Interface layer\nâ”œâ”€â”€ src/test/               # Test code\nâ””â”€â”€ build.gradle           # Build configuration\n```\n\n### Frontend Structure\n```\ncmc-frontend/               # CMC Management System\nâ”œâ”€â”€ src/\nâ”‚   â”œâ”€â”€ components/        # React components\nâ”‚   â”œâ”€â”€ pages/            # Next.js pages\nâ”‚   â””â”€â”€ styles/           # Style files\nâ””â”€â”€ package.json\n\nconsumer-frontend/          # Consumer Application\nâ”œâ”€â”€ src/\nâ”‚   â”œâ”€â”€ app/              # Angular application\nâ”‚   â”œâ”€â”€ components/       # Angular components\nâ”‚   â””â”€â”€ services/         # Angular services\nâ””â”€â”€ package.json\n```\n\n## First Contribution\n\n### 1. Create Feature Branch\n```bash\ngit checkout -b feature/your-feature-name\n```\n\n### 2. Write Code\n- Follow [Coding Standards](../coding-standards.md)\n- Write unit tests\n- Update related documentation\n\n### 3. Run Quality Checks\n```bash\n# Run all tests\n./gradlew test\n\n# Code quality check\n./gradlew check\n\n# Format code\n./gradlew spotlessApply\n```\n\n### 4. Commit Changes\n```bash\ngit add .\ngit commit -m \"feat: add new feature description\"\ngit push origin feature/your-feature-name\n```\n\n### 5. Create Pull Request\n- Use clear title and description\n- Link related Issues\n- Ensure all checks pass\n- Request code review\n\n## Development Workflow\n\n### Daily Development\n1. **Pull Latest Code**: `git pull origin main`\n2. **Create Feature Branch**: `git checkout -b feature/xxx`\n3. **TDD Development**: Write tests first, then implementation\n4. **Code Review**: Submit PR and request review\n5. **Merge Code**: Merge to main branch after review approval\n\n### Testing Strategy\n- **Unit Tests**: Test business logic\n- **Integration Tests**: Test component interactions\n- **End-to-End Tests**: Test complete processes\n\n## Common Issues\n\n### Q: How to resolve build failures?\nA: Check if Java version is 21, clean build cache: `./gradlew clean build`\n\n### Q: How to set up IDE?\nA: Refer to [Development Tools Configuration](../tools-and-environment/README.md)\n\n### Q: How to run specific tests?\nA: Use `./gradlew test --tests \"ClassName.methodName\"`\n\n### Q: How to connect to development database?\nA: Development environment uses H2 by default, viewable at `http://localhost:8080/h2-console`\n\n## Learning Resources\n\n### Internal Documentation\n- [Architecture Design](../architecture/README.md)\n- [Testing Guide](../testing/README.md)\n- [API Design](../coding-standards/api-design-standards.md)\n\n### External Resources\n- Spring Boot Official Documentation\n- React Official Documentation\n- Angular Official Documentation\n\n## Getting Help\n\n### Team Contact\n- **Technical Questions**: Ask in Slack #dev-help channel\n- **Architecture Discussion**: Contact architecture team\n- **Urgent Issues**: Contact technical lead\n\n### Documentation Contribution\nIf you find documentation errors or need improvements, feel free to:\n1.
