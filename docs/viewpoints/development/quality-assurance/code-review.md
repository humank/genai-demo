# 程式碼審查指南

## 審查流程

### 提交前檢查
- 程式碼格式化
- 單元測試通過
- 靜態分析無錯誤

### 審查標準
- 程式碼品質
- 設計原則
- 安全性檢查
- 效能考量

## 審查清單

### 功能性
- [ ] 需求實作正確
- [ ] 邊界條件處理
- [ ] 錯誤處理完整

### 可讀性
- [ ] 命名清晰
- [ ] 註解適當
- [ ] 結構清楚

### 可維護性
- [ ] 模組化設計
- [ ] 低耦合高內聚
- [ ] 遵循 SOLID 原則

## 工具支援

### 自動化檢查
- SonarQube
- Checkstyle
- SpotBugs
- PMD

### 審查平台
- GitHub Pull Request
- GitLab Merge Request
- Bitbucket Code Review
